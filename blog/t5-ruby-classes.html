Glasses Classes

My real world example of Ruby Classes.

May 31, 2015

Week 5 of Phase 0 at DBC has mostly consisted on classes. A class is said to be a blueprint of an object. You can assign methods to a class and then any object that falls within that class with be endowed with the corresponding methods. Therefore, a class allows you to bundle methods so that you can quickly make objects that behave similarly. This will save you a lot of time and typing.

In this blog, I will model a real world object, glasses, as a class. When modeling classes we want to consider two things: what the object knows, which are known as instance variables, and, what the object does, which are known as instance methods. Instance variables represent the object's many states. In the example of glasses, some instance methods would include lens, frame, etc. Some instance methods would include wear, lose or sit on.
