Github Cheat Sheet!

Starting Phase 0 soon? A handy guide on using Github!

Git is a local program that tracks different changes on your computer. Github is remote and is stored on the internet. A real world comparison would be Git as a flash drive and Github as dropbox. Throughout your time with Dev Bootcamp you will be become very familiar with the two. Below are some of the common processes that you will have to execute week by week.

Forking a repository means you are copying it from one Github repository(often times Dev Bootcamp's) to your Github on the internet.
How to Fork from Github
Step 1. Click the fork button on the top right of the repo page that you want to fork. This makes a copy of it on your github.
Step 2. If you want others to comment on your repo click "enable issues" on the settings.

Cloning means you are taking the copied repository, or repo, from Github and downloading it onto your computer.
How to Clone from Github
Step 1. On your github look for the ssh link on the right middle of the page. Copy that url of your repo.
Step 2. Then in your terminal type git clone http://.........

Pushing to Github means that you are taking any changes you have made locally on your computer and are pushing, or saving, those changes to the version you have on the internet.
How to Push to Github
Step 1. Make sure you are on the Master Branch.
Step 2. Make sure you're in the folder directory that you want to be in.
Step 3. Git Add . (or just add the files that you want to add)
Step 4. Git Commit -m "write a sentence about what you did"
Step 5. Git Push Origin Master

Fetching changes means that you are taking the changes made to the version on the internet and are pulling them down to your computer. Merging takes those fetched changes and combines them with the changes you have already made on your computer. Merging will not override your changes.
How to Fetch & Merge Changes from Github
Step 1. Check your remotes (Git remote-v)
Step 2. Make sure your upstream is linked to the repo that you want to pull from.
Step 3. If it is linked Skip to step 5 if not continue to step 4.
Step 4. Type "Git remote add upstream" in your terminal. Then go to github and copy the url of the place you want to pull from and add .git to the end of the url. The url gets pasted after the "Git remote add upstream" so a final copy would look like this : git remote add upstream http://devbootcamp.com/phase-0-unit-1.git
Step 5. Type git fetch upstream
Step 6. git merge upstream/master
Step 7. git commit -m "write a sentence about what you did"
Step 8. git push origin master
